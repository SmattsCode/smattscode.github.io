# Referential Integrity
Ensures that relationships between tables remain consistent and accurate.  It guarantees that a [[Database/Terms/Foreign Key\|Foreign Key]] in one table always is null or refers to only one valid [[Database/Terms/Primary Key\|Primary Key]] in another, thus preventing "orphaned" records.  For example a Sales Invoice with a non existent Customer.
